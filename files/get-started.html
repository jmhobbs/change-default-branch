Before we begin, let's talk about what is going to happen here.

For each repository you choose to convert, the following will happen:

1. If the new branch name you have chosen does not exist, the tool will create it.

2. Once the branch exists, the tool will update the repository and set the default branch to the one you have chosen.

Currently it does not remove any master branches from your repository.

To accomplish this, we have to ask for very broad permissions on Github. If we had a way to do it with less, we would.

Changing your default branch can be problematic for PR's, forks, and CI/CD tools. That's ok, and worth the effort, but you should be aware there may be tweaks required after you change over.
